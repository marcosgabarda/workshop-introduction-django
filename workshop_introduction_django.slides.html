¿Qué es Django?

Es un framework para el desarrollo de aplicaciones Web, y se define a si mismo como:

El Web Framework para perfeccionistas con las fechas de entrega

Con Django tenemos:

  • Un sistema de mapeo objeto-relacional (ORM)
  • Diseño y enrutamiento de URLs
  • Lenguaje propio de templates
  • Formularios reutilizables
  • Y muchas cosas más

Pero sobretodo...

  • Django es Python

El patrón MVC y los deadlines

El patrón Modelo-Vista-Controlador es más extendido en el desarrollo de aplicaciones Web:

MVC

  • Modelo: Se encarga del acceso a los datos y de la lógica de negocio
  • Vista: Muestra la información al usuario
  • Controlador: Maneja las interacción del usuario con la aplicación

Partiendo de esta base, Django implementa un patrón que llama Modelo-Template-Vista, buscando efatizar el desarrollo rápido.

  • Vista se correspondería con el controlador
  • Template se correspondería con la vista

Estructura de un proyecto en Django

Cuando realizamos una aplicación web basada en Django es importante tener claro estos conceptos:

  • Django project que será la aplicación web desarrollada usando este framework
  • Django app, pequeñas librerías diseñadas para representar una parte del proyecto
  • Third-party package, Django apps reutilizables que han desarrollado otras personas y hemos instalado con pip

Historias de usuario

  1. Un visitante podrá registarte en nuestrá página con un nombre de usuario y contraseña
  2. Un visitante podrá identificarse en la página como un usuario usando los datos del registro
  3. Un usuario podrá indicar que se ha tomado un café, seleccionandolo de una lista
  4. Un usuario podrá agregar un nuevo café a la lista, indicando un nombre, una imágen y una descripción

Los diferentes elementos de un proyecto recién creado:

  • Carpeta coffeproject en la raíz es el contenedor del proyecto

  • Sub-carpeta coffeeproject es el paquete de Python principal para el proyecto
  • coffeeproject/settings.py contiene la configuración del proyecto
  • coffeeproject/urls.py contiene las declaraciones de las URLs del proyecto
  • coffeeproject/wsgi.py punto de entrada para servidores compatibles con WSGI (para la puesta en producción)

models.py

Crearemos los modelos para poder representar un tipo de café y la acción "un usuario se toma un café".

In []: 
from django.conf import settings
from django.db import models

# Create your models here.


class Coffee(models.Model):
    """
    A coffee type.
    """
    name = models.CharField(max_length=128)
    picture = models.ImageField(upload_to="pictures", null=True, blank=True, default='/static/img/coffee.jpg')

    creator = models.ForeignKey(settings.AUTH_USER_MODEL, related_name="coffees_created", null=True, blank=True)
    created_at = models.DateTimeField(auto_now_add=True)


class CoffeeTaken(models.Model):
    """
    User drinks coffee.
    """
    coffee = models.ForeignKey(Coffee, related_name="users_taken")
    user = models.ForeignKey(settings.AUTH_USER_MODEL, related_name="coffees_taken")

    created_at = models.DateTimeField(auto_now_add=True)

views.py

Mostraremos los detalles de un tipo de café y un listado de los cafés disponibles

In []: 
from django.shortcuts import render, get_object_or_404
from coffees.models import Coffee


def coffee_details(request, coffee_id):
    """
    Details of a coffee
    """
    coffee = get_object_or_404(Coffee, pk=coffee_id)
    return render(request, "coffees/details.html", {"coffee": coffee})


def coffees_list(request):
    """
    List of available coffess
    """
    coffees = Coffee.objects.all().order_by("-created_at")
    return render(request, "coffees/list.html", {"coffees": coffees})

Actualizamos las vistas para añadir la que se encargará de manejar el formulario:

In []: 
from django.core.urlresolvers import reverse
from django.shortcuts import render, get_object_or_404, redirect
from coffees.forms import CreateCoffee
from coffees.models import Coffee

# ...

def coffee_create(request):
    """
    Creates new type of coffee.
    """
    if request.method == "GET":
        form = CreateCoffee()
    else:
        form = CreateCoffee(request.POST)
        if form.is_valid():
            coffee = form.save()
            return redirect(reverse('coffee_details', kwargs={'coffee_id': coffee.pk}))
    return render(request, "coffees/new.html", {"form": form})

Ficheros subidos por los usuarios

Para hacer accesibles los ficheros que los usuarios suben, hay que hacerlos accesibles en el fichero de URLs.

*Esto no debe hacerse en un entorno de producción, donde los ficheros estáticos deberían ser accedidos usando un servidor como Nginx

In []: 
from django.conf.urls import patterns, url
from django.conf import settings
from django.conf.urls.static import static
from django.contrib import admin
admin.autodiscover()

urlpatterns = patterns(
    '',
    url(r'^$', "coffees.views.coffees_list", name="coffees_list"),
    url(r'^coffees/(?P<coffee_id>\d+)/', "coffees.views.coffee_details", name="coffee_details"),
    url(r'^coffees/new/', "coffees.views.coffee_create", name="coffee_create"),
) + static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)
